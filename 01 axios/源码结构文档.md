# Axios 源码结构文档

## 项目概述

**项目名称**: axios
**版本**: 1.14.0
**架构类型**: 模块化多文件架构 (Modular Multi-File)
**许可证**: MIT

Axios 是一个基于 Promise 的 HTTP 客户端，用于浏览器和 Node.js 环境。它支持发送 HTTP 请求、拦截器、取消请求、请求/响应转换等高级功能。

### 2. 请求流程

```
请求发起
    │
    ▼
┌──────────────────────────────────────┐
│ 1. Axios.request(config)            │
│    - 参数标准化                       │
│    - 配置合并 (mergeConfig)           │
└──────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────┐
│ 2. 拦截器预处理 (Request Interceptors)│
│    - 等待所有请求拦截器执行完毕        │
└──────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────┐
│ 3. dispatchRequest(config)           │
│    - 数据转换 (transformData)        │
│    - Content-Type 设置               │
└──────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────┐
│ 4. Adapter (适配器选择)               │
│    - xhr (浏览器 XMLHttpRequest)     │
│    - http (Node.js http 模块)        │
│    - fetch (Fetch API)               │
└──────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────┐
│ 5. 响应拦截器 (Response Interceptors) │
│    - 响应数据转换                     │
└──────────────────────────────────────┘
    │
    ▼
返回 Promise<Response>
```

## 核心模块详解

### 1. axios.js (入口模块)

**路径**: `lib/axios.js` (~118行)

**职责**: 创建 axios 实例，导出所有公共 API

```javascript
// 核心逻辑
function createInstance(defaultConfig) {
  const context = new Axios(defaultConfig);
  const instance = bind(Axios.prototype.request, context);

  // 复制原型方法到实例
  utils.extend(instance, Axios.prototype, context, { allOwnKeys: true });
  // 复制实例属性
  utils.extend(instance, context, null, { allOwnKeys: true });

  // 添加工厂方法
  instance.create = function create(instanceConfig) {
    return createInstance(mergeConfig(defaultConfig, instanceConfig));
  };

  return instance;
}
```

**导出内容**:

- `axios` - 默认实例
- `axios.Axios` - 核心类
- `axios.CancelToken` - 取消令牌
- `axios.CanceledError` - 取消错误
- `axios.isCancel` - 判断取消状态
- `axios.VERSION` - 版本号

### 2. Core/Axios.js (核心类)

**路径**: `lib/core/Axios.js` (~316行)

**职责**: HTTP 请求的核心逻辑，拦截器管理

**主要方法**:

| 方法 | 作用 |
|------|------|
| `request(configOrUrl, config)` | 主请求方法 |
| `get(url, config)` | GET 请求 |
| `post(url, data, config)` | POST 请求 |
| `put(url, data, config)` | PUT 请求 |
| `delete(url, config)` | DELETE 请求 |
| `patch(url, data, config)` | PATCH 请求 |
| `head(url, config)` | HEAD 请求 |
| `options(url, config)` | OPTIONS 请求 |

**拦截器机制**:

```javascript
this.interceptors = {
  request: new InterceptorManager(),  // 请求拦截器
  response: new InterceptorManager(), // 响应拦截器
};
```

### 3. Core/dispatchRequest.js (请求分发)

**路径**: `lib/core/dispatchRequest.js` (~101行)

**职责**: 调用适配器发送请求，处理数据转换

**关键流程**:

1. 检查取消状态
2. 转换请求数据 (`transformData`)
3. 设置默认 Content-Type
4. 获取适配器并发送请求
5. 转换响应数据

### 4. Adapters (适配器系统)

#### 4.1 adapters.js (适配器解析)

**路径**: `lib/adapters/adapters.js` (~153行)

**职责**: 根据环境选择合适的适配器

**适配器优先级**:

```javascript
adapter: ['xhr', 'http', 'fetch']
```

#### 4.2 xhr.js (浏览器适配器)

**路径**: `lib/adapters/xhr.js` (~333行)

**功能特性**:

- 基于 XMLHttpRequest
- 进度事件支持 (上传/下载)
- 取消机制 (CancelToken + AbortSignal)
- 超时处理
- 响应类型支持 (text/json/arraybuffer/blob/document)
- 跨域支持

#### 4.3 http.js (Node.js 适配器)

**路径**: `lib/adapters/http.js`

**功能特性**:

- 基于 Node.js `http`/`https` 模块
- 自动解压 (gzip/deflate)
- Cookie 处理
- 代理支持
- HTTPS 证书验证

#### 4.4 fetch.js (Fetch API 适配器)

**路径**: `lib/adapters/fetch.js`

**功能特性**:

- 基于原生 Fetch API
- 支持 AbortController
- 流式响应处理

### 5. InterceptorManager.js (拦截器管理)

**路径**: `lib/core/InterceptorManager.js` (~78行)

**核心方法**:

| 方法 | 作用 |
|------|------|
| `use(fulfilled, rejected, options)` | 添加拦截器 |
| `eject(id)` | 移除拦截器 |
| `clear()` | 清空所有拦截器 |
| `forEach(fn)` | 遍历拦截器 |

**执行顺序**:

```
请求拦截器: 1 → 2 → 3 (后添加的先执行)
     ↓
  实际请求
     ↓
响应拦截器: 1 → 2 → 3 (先添加的先执行)
```

## Cancel (取消机制)

### 两种取消方式

#### 1. CancelToken (传统方式 - 已废弃)

```javascript
const CancelToken = axios.CancelToken;
const source = CancelToken.source();

axios.get('/user/123', {
  cancelToken: source.token
});

source.cancel('Operation canceled');
```

#### 2. AbortController (现代方式 - 推荐)

```javascript
const controller = new AbortController();

axios.get('/user/123', {
  signal: controller.signal
});

controller.abort();
```

## 技术特点

### 1. 适配器模式

```javascript
// 根据环境自动选择适配器
const adapter = adapters.getAdapter(config.adapter || defaults.adapter, config);
return adapter(config);
```

### 2. 拦截器链

```javascript
// 请求拦截器 (后进先出)
dispatchRequest = chain.shift()(dispatchRequest, chain.pop());

// 响应拦截器 (先进先出)
response = chain.shift()(response, undefined);
```

### 3. 数据转换

```javascript
// 请求前
config.data = transformData.call(config, config.transformRequest);

// 响应后
response.data = transformData.call(config, config.transformResponse, response);
```

## 总结

Axios 采用**模块化分层架构**：

1. **入口层**: 统一的 ES Module / CommonJS 导出
2. **实例层**: axios 实例工厂
3. **核心层**: Axios 类 + 拦截器 + 请求分发
4. **适配器层**: XHR / HTTP / Fetch 三种适配器
5. **平台层**: 浏览器 / Node.js 特定实现

这种架构的优势：

- **环境适配**: 同一套 API 支持浏览器和 Node.js
- **可扩展性**: 适配器模式便于添加新的 HTTP 客户端
- **可测试性**: 模块化设计便于单元测试
- **按需加载**: 支持树摇优化，减小打包体积
